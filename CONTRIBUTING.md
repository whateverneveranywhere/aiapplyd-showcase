# Contributing

This is the public marketing + docs repo. The product source code is private.

What you can contribute here:

---

## Bug reports for the public site

Found something broken on aiapplyd.com? Open an issue.

- Use the [bug report template](.github/ISSUE_TEMPLATE/bug-report.md)
- Include: page URL, what you expected, what happened, browser + OS, screenshot if relevant
- For payment / account issues, email hello@aiapplyd.com instead, faster path

---

## FAQ corrections

If something in [FAQ.md](./FAQ.md) is wrong or out of date:

- Open a PR with the correction
- One PR per fix is fine, batched is also fine
- Cite a source if the claim is factual (pricing, callback rates, etc.)

---

## Feature requests

Open an issue using the [feature request template](.github/ISSUE_TEMPLATE/feature-request.md).

Required:
- The actual job-search problem you're trying to solve (not the implementation you want)
- A use case from your own search if you have one
- Whether you'd pay for it as a separate add-on or expect it in the existing plans

We read every one. We ship the ones that hit a real problem multiple users have.

---

## Blog post topics

If there's something you want me to write about (a teardown of a competitor, a deep-dive on how a specific ATS works, a tactic that worked for you), open an issue tagged `blog-idea`.

The most useful posts come from real questions, not topic generators.

---

## Things we don't accept here

- Pull requests against the product source (it's private, this repo doesn't have it)
- Resume reviews (use the product, that's literally what it does)
- "Add my tool to your comparison table" without data (bring callback rates, pricing, ban-risk evidence)
- Generic SEO link requests

---

## Style

When you write in this repo:

- No em-dash characters
- No "passionate about", "thrilled", "leverage" (verb), "robust", "seamless"
- Direct sentences, blunt facts, real numbers when you have them
- If you don't know something, say "I don't know" instead of guessing

---

## Code of conduct

Be civil. Don't be a jerk to other contributors. Disagreements about the product are welcome, ad hominem isn't.

If someone violates this, email hello@aiapplyd.com.
